 ## Employee Audit Service / Сервис аудита сотрудников

 Employee Audit Service — это консольное приложение на Spring Framework, созданное для эффективного управления 
 ресурсами компании через удобный аудит сотрудников. Проект демонстрирует профессиональный подход к разграничению прав доступа 
 и чистую архитектуру с использованием современных паттернов.

## О проекте

Программа решает важную бизнес-задачу — контроль и распределение кадровых ресурсов с учетом ролевой модели. 
В системе предусмотрены четыре уровня доступа:

- Администратор — полный контроль над системой
- HR — управление кадровыми данными
- Менеджер — ограниченное управление командой
- Регулярный пользователь — просмотр базовой информации

Каждая позиция имеет строго разграниченные права доступа, что обеспечивает безопасность и целостность данных.

## Ключевые особенности

### 🎯 Основной функционал
Управление сотрудниками через стандартные операции:
- Создание нового сотрудника
- Редактирование данных сотрудника
- Получение информации о конкретном сотруднике
- Удаление сотрудника из системы
- Просмотр всех сотрудников

### ✨ Архитектурные решения

#### 🔒 АОП (Аспектно-ориентированное программирование) для валидации и логирования
Главная "фишка" проекта — реализация сквозных аспектов через Spring AOP (на Java-конфигурации):
- Валидация прав доступа — проверка, имеет ли пользователь право на выполнение операции
- Аудит действий — автоматическое логирование всех действий пользователей

Аспекты не вмешиваются в бизнес-логику, но обеспечивают критически важную функциональность, следуя принципу разделения ответственности.

#### 🏗 Паттерн Command
Для упрощения сервисного слоя и улучшения читаемости кода применен паттерн Command:
- Каждая операция инкапсулирована в отдельную команду
- Упрощено добавление нового функционала
- Код стал более структурированным и тестируемым

## Технологический стек

| Компонент | Технология |
|-----------|------------|
| Язык | Java 21 |
| Фреймворк | Spring Framework (классический, без Spring Boot) |
| Аспекты | Spring AOP (Java-конфигурация) |
| Паттерн | Command |
| Сериализация | Jackson |
| Сборка | Maven |

## Гибкость и расширяемость

Программа создана с прицелом на будущее развитие. Фундамент уже заложен, поэтому легко можно:

- 🔌 Подключить базу данных (сейчас, вероятно, используется in-memory хранение)
- 📦 Добавить новые сущности (отделы, проекты, зарплатные ведомости)
- 🔐 Расширить ролевую модель
- 🌐 Перевести на веб (REST API поверх существующей логики)
- 📊 Добавить аналитические отчеты

> 💡 Проект не предназначен для production, но отлично демонстрирует возможности разработчика: чистая архитектура,
> грамотное применение паттернов и АОП, разделение ответственности.
